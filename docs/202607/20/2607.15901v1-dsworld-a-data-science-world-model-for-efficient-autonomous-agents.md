## 摘要
尽管自主数据科学代理在数据理解和决策方面能力强大，但它们仍严重依赖于涉及昂贵计算的试错工作流。这一瓶颈促使模型能够在实际执行之前预测数据科学操作的效果。本文引入了数据科学世界模型的概念，该模型通过基于当前工作流状态和候选操作预测环境状态转移来模拟数据科学执行环境。我们进一步提出了DSWorld，一个实用框架，结合了结构化状态构建、成本感知路由、轻量级实际执行以及用于昂贵操作的基于LLM的模拟器。为支持训练，我们构建了8K规模的转移轨迹数据集，并引入了反思性世界模型优化，一种用于改进转移预测的误差感知强化学习策略。实验表明，DSWorld将基于强化学习的代理训练加速约14倍，将基于搜索的推理加速约3-6倍，同时保持竞争性能，并在转移预测任务上比最强LLM基线高出35.6%。代码可在https://anonymous.4open.science/r/DSWorld获取。

## Abstract
Despite strong capabilities in data understanding and decision-making, autonomous data science agents still heavily rely on trial-and-error workflows that involve expensive computation. This bottleneck motivates models that can anticipate the effects of data science operations before real execution. In this paper, we introduce the concept of Data Science World Model, which model the data science execution environment by predicting environment state transitions conditioned on current workflow states and candidate operations. We further propose DSWorld, a practical framework that combines structured state construction, cost-aware routing, lightweight real execution, and an LLM-based simulator for expensive operations. To support training, we construct an 8K-scale transition trajectory dataset and introduce Reflective World Model Optimization, an error-aware reinforcement learning strategy for improving transition prediction. Experiments show that DSWorld accelerates RL-based agent training by approximately $14\times$ and search-based inference by approximately $3$-$6\times$ while maintaining competitive performance, and outperforms the strongest LLM baseline by 35.6% on transition prediction tasks. The code is available at https://anonymous.4open.science/r/DSWorld.

## 1. 核心问题与整体含义（研究动机与背景）
- **问题**：当前自主数据科学代理（如ML-Master、AIDE等）依赖昂贵的计算密集型试错工作流，大量执行时间消耗在数据处理、模型训练和评估上，而非代理本身的推理。例如，ML-Master在MLE-Bench中超过86%的执行时间用于模型训练。这严重限制了系统的效率和可扩展性。
- **核心动机**：能否开发一个**数据科学世界模型**，使代理在执行操作前就能预测其对环境状态的影响（如数据变化、模型性能、错误信息等），从而避免不必要的实际计算。
- **整体含义**：DSWorld首次将世界模型（World Model）的概念引入数据科学领域，为自主数据科学代理提供一种高效的“预测-执行”混合机制，显著加速训练和推理过程。

## 2. 方法论：核心思想、技术细节与流程
- **核心思想**：将数据科学执行环境视为一个“世界”，给定当前工作流状态 \( S_t \) 和候选操作 \( A_t \)，模型预测下一个状态 \( S_{t+1} \)。通过结合真实执行（Compiler）和LLM模拟（Simulator），在保证准确性的同时提升效率。
- **框架组成**：DSWorld由四个组件构成：`W = {SC, R, C, S}`
  - **状态构造器（SC）**：将原始环境转换为结构化状态表示，包含任务、数据、执行环境、历史日志等。
  - **路由器（R）**：对动作进行嵌入编码，并基于成本决定是真实执行（mt=execute）还是模拟（mt=simulate）。轻量级动作（如简单数据操作）走编译器，昂贵动作（如大模型训练）走模拟器。同时设置了超时机制：若编译器执行超时，则回退到模拟器。
  - **编译器（C）**：对轻量级动作直接在实际环境中执行，返回真实的下一个状态。
  - **模拟器（S）**：基于LLM（Qwen3-8B）预测下一个状态，避免实际计算。模拟器经过两阶段训练：SFT初始化 + 反思式RL优化。
- **优化策略——反思式世界模型优化（RWMO）**：
  1. 初始SFT：在转移数据上微调模拟器。
  2. 强化学习：对每个样本执行n次rollout，得到原始预测和基于反思反馈的修正预测。通过GRPO（Group Relative Policy Optimization）进行训练，组内优势函数计算奖励标准化，奖励函数包括执行状态、输出、错误、性能等维度的评估。
  3. 反思机制：比较预测与真实状态，生成反馈，再基于反馈修正预测，形成迭代改进。
- **训练数据构建——DSWorld-8K**：
  - **真实轨迹收集**：基于现有代理和基准（如DA-Code）运行获得真实环境转移。
  - **合成轨迹构建**：基于MMTU数据集（60K真实表格）生成多样化环境状态；利用NumPy/Pandas操作库和错误类型库，由LLM合成可执行动作；通过编译器实际执行获得真实下一个状态，并验证一致性。同时为每个转移生成Chain-of-Thought推理轨迹。
  - 最终数据集包含约8000个样本及其推理轨迹。

## 6. 主要结论与发现
- DSWorld在**转移预测**上平均超越最强LLM基线o4-mini达**35.6%**，尤其在执行相关任务上提升显著（ESP提升33.4%，ETP 57.6%，ERS 71.5%，EKM 50.5%）。

- 在**代理推理**中，DSWorld实现**3-6倍加速**，同时保持或略降性能，而DeepSeek 3.2导致严重性能下降。
- **反思式世界模型优化**有效：完整DSWorld相比仅GRPO再提升1.3%平均性能。
- **数据和模型规模扩展**持续提升性能：训练数据从0.1k增至6.4k，模型从0.6B增至14B，均带来显著增益。
- **跨环境泛化**表现一致，Ubuntu、CentOS、Windows下性能波动很小。

## 7. 优点

- **实验全面**：不仅评估转移预测能力，还验证了在代理训练和推理中的实际加速效果，并做了消融、规模、跨环境分析。

## 8. 不足与局限
- **未建模外部工具调用**：当前DSWorld只建模数据科学操作，未涵盖如API调用、文件系统操作等外部工具对状态的影响。
- **模拟器能力上限**：基于LLM的模拟器在复杂、多步工作流场景下仍可能产生不准确预测（作者已指出）。
- **合成数据分布偏差**：合成轨迹与真实自主代理工作流之间存在分布差距，可能限制泛化（如附录C案例显示某些数据预览报错，但DSWorld预测执行成功，存在潜在不一致）。

- **计算资源**：仅报告使用4块A800，未提及训练总时长，且RL仅200步，可能未达到最优。
- **对比基线**：训练后基线仅使用相同骨干（Qwen3-8B）的SFT/GRPO，未与其他专门领域模型或更大规模模型（如DSWorld + 14B）进行直接性能比较。
