## 摘要
世界动作模型（WAMs）通过基于视频的世界建模为机器人动作学习提供密集监督，成为视觉-语言-动作（VLA）策略的一种有前景的替代方案。现有WAM擅长物理基础执行，但通常缺乏基于VLM的VLA中用于分解粗略指令的显式语言级规划接口。当家庭任务涉及复杂多步目标时，这种分解变得重要，因为粗略的用户命令需要转化为一系列细粒度的可执行子任务。同时，由于现有系统在数据、机器人形态和任务协议上存在差异，该领域仍缺乏VLA与WAM执行能力的公平真实机器人对比。为解决分解差距以及受控的WAM-VLA对比需求，我们提出了DSWAM，一个面向细粒度机器人操作的双系统世界动作基础模型。DSWAM保留系统1 WAM执行器作为默认控制路径，并仅在任务分解有益时可选激活系统2视觉语言子任务规划器。规划器从短期视觉历史和全局任务提示中预测可执行子任务，而WAM执行器对每个指令或子任务执行具有世界感知的动作生成。执行器通过动作预测和视频联合训练，但推理时直接预测动作块，无需显式未来视频生成。为使该执行路径在真实机器人上实用，我们进一步集成了TensorRT加速、异步执行和实时分块（RTC），使策略查询不阻塞机器人控制。为与VLA策略进行公平的真实机器人对比，我们在DeMaVLA真实世界可变形操作设置下构建和评估DSWAM，匹配了机器人平台、预训练数据、后训练数据和评价标准。

## Abstract
World Action Models (WAMs) provide a promising alternative to Vision-Language-Action (VLA) policies by using video-based world modeling as dense supervision for robot action learning. Existing WAMs excel at physically grounded execution, but typically lack the explicit language-level planning interface in VLM-based VLAs for decomposing coarse instructions. Such decomposition becomes important when household tasks involve complex multi-step goals, where coarse user commands need to be converted into sequences of fine-grained executable subtasks. Meanwhile, the field still lacks a fair real-robot comparison between VLA and WAM execution capabilities, since existing systems often differ in data, robot embodiments, and task protocols. To address both the decomposition gap and the need for a controlled WAM-VLA comparison, we introduce DSWAM, a Dual-System World Action Foundation Model for fine-grained robot manipulation. DSWAM keeps a System 1 WAM executor as the default control path and optionally activates a System 2 vision-language subtask planner only when task decomposition is useful. The planner predicts executable subtasks from short-term visual history and a global task prompt, while the WAM executor performs world-aware action generation for each instruction or subtask. The executor is trained with action prediction and video co-training, but inference directly predicts action chunks without explicit future video generation. To make this execution path practical on real robots, we further integrate TensorRT acceleration, asynchronous execution, and real-time chunking (RTC) so that policy queries do not block robot control. To provide a fair real-robot comparison with VLA policies, we build and evaluate DSWAM under the DeMaVLA real-world deformable manipulation setting with matched robot platform, pretraining data, post-training data, and evaluation criteria.

- **核心问题**：如何同时实现：
  1. 物理基元执行（WAM 擅长）；
  2. 需要时的语言级任务分解（VLA 擅长）；
  3. 在受控条件下公平对比 WAM 与 VLA 策略的执行能力。
- **本文贡献**：提出 **DSWAM**（Dual-System World Action Foundation Model），采用双系统架构——System 1 为默认 WAM 执行器，System 2 为可选的视觉语言子任务规划器，仅在分解有益时激活。在匹配机器人平台、数据、任务协议和评估标准的 DeMaVLA 设定下，与 VLA 公平对比，验证了 WAM 式执行在可变形操作中的优势。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- 将**语义任务组织**与**物理执行**解耦：System 1 专攻基于世界的动作生成，System 2 负责粗粒度指令的分解，两者通过语言指令接口耦合。
- 训练时利用视频联合监督学习动态，推理时直接输出动作块，避免显式未来视频生成的推理开销。

### 2.2 关键技术细节
- **观测表示**（式1）：多视图 RGB 图像 \(I_t^{1,2,3}\)、语言指令 \(\ell_t\)、本体感受 \(q_t\) 构成当前上下文 \(O_t\)。
- **动作块**（式2）：预测长度为 \(H\) 的连续动作序列 \(A_t\)。
- **System 2 子任务规划器**：
  - 基于 Rynnbrain4B 风格的视觉语言模型。
  - 输入：短期视觉历史 \(X_t\)（最近 5 帧，1 Hz 采样） + 全局任务提示 \(p\)。
  - 输出：下一个子任务指令 \(s_t = P_{\phi}(X_t, p)\)。
  - 训练数据由机器人轨迹视频自动分割得到，采用过渡感知监督（式6）：对于当前窗口，若在子任务内部则输出当前子任务；若在边界则输出下一子任务；若在终止子任务末尾则输出“done”。
  - 优化目标：自回归语言建模损失 \(L_{\text{plan}}(\phi)\)（式7）。
- **System 1 WAM 执行器**：
  - 编码器 \(f_\theta\) 将当前 \(O_t\) 映射为潜在世界特征 \(z_t\)（式9）。
  - 动作头基于**条件流匹配**（Conditional Flow Matching）直接预测动作块 \(A_t\)（式10）。
  - 训练联合两个流匹配损失（式12-15）：
    - 动作损失 \(L_{\text{act}} = \text{LFM}(A_t)\)；
    - 视频联合训练损失 \(L_{\text{vid}} = \text{LFM}(V_{t+1:t+T})\)；
    - 总损失 \(L = L_{\text{act}} + \lambda_{\text{vid}} L_{\text{vid}}\)。
  - 推理时：只执行动作分支，不进行未来视频生成。
- **部署优化**：
  - **TensorRT 加速**：拆分为视觉上下文引擎和动作去噪引擎，推理延迟从 198.2 ms 降至 73.8 ms（RTX 5090，BF16）。
  - **实时分块（RTC）** 和**异步执行**：策略查询不阻塞控制循环，解耦推理与执行。

## 6. 主要结论与发现

1. **真实折叠任务**：DSWAM（WAM-only）平均成功率达 **96.3%**（DeMaVLA 92.5%），完成时间从 2'18" 缩短至 1'44"，尤其在裤子任务上提升显著（SR 从 75%→90%）。
2. **仿真 RoboTwin 2.0**：DSWAM 平均 SR 清洁 92.38%、随机 91.90%，全面优于所有对比方法（包括 Fast-WAM 91.88%/91.78%）。
3. **System 2 有效性**：子任务级指令在排序任务上达到 100% SR，而原始粗指令仅 75.7%，且执行错误从 3.53 降至 0.65。
4. **部署效率**：TensorRT 加速 2.69 倍，异步 RTC 进一步提升执行稳定性（裤子任务 SR 从 70% 升至 100%）。
5. **核心启示**：WAM 式执行在可变形操作中可作为强基线，配合可选的语义规划器，可同时实现高效物理执行与复杂任务组织。

## 7. 优点

- **架构创新**：双系统解耦使系统在不同粒度指令下都能发挥优势，且 System 2 为可选，不增加原子指令不必要的计算。
- **公平对比**：通过 DeMaVLA 协议匹配所有实验变量，首次在受控条件下直接比较 WAM 与 VLA 执行能力，结论可信度高。
- **训练-推理分离**：视频联合训练提供丰富时空监督，而推理时消除生成步骤，兼顾学习效果与实时性。
- **工程实用**：集成 TensorRT、RTC、异步执行，降低部署延迟，提升闭环稳定性。
- **全面评估**：仿真+真实、多基线、消融及效率分析，实验设计较为完整。

## 8. 不足与局限

- **实验覆盖有限**：
  - System 2 仅在单一种排序任务上验证，缺乏在多步操作（如完整折叠流程）或有歧义指令下的泛化测试。
  - 真实世界任务类型单一（折叠和简单排序），未涵盖抓取、放置、移动等更广泛家庭操作。
- **泛化性未知**：模拟 RoboTwin 仅 50 个任务，且未测试零样本迁移或从未见过的物体/场景。
- **算力与可复现性**：未报告训练阶段 GPU 型号、数量、时长，增加了复现难度；数据收集需 5,000+ 小时真实机器人数据，成本高。
